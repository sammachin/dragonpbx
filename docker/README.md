# DragonPBX Docker Deployment

Docker Compose setup for running DragonPBX and all its dependencies.

## Services

| Service | Description | Port |
|---------|-------------|------|
| **dragonpbx** | SIP application server | 2999 |
| **dragonpbx-ui** | Web management UI | 1337 |
| **drachtio** | SIP server (drachtio) | 5060 (UDP/TCP) |
| **rtpengine** | RTP media proxy | 2223, 40000-40100 (UDP) |
| **redis** | Session/config cache (internal only) | - |

## Quick Start

```bash
./start.sh
```

This will:
1. Generate a random API token in `.env` (first run only)
2. Build and start all containers

Alternatively, run manually:

```bash
# Generate a token
echo "API_TOKEN=$(openssl rand -hex 32)" > .env

# Start services
docker compose up
```

## Configuration

Environment variables are set in `.env`. The file is auto-generated by `start.sh` on first run.

| Variable | Default | Description |
|----------|---------|-------------|
| `API_TOKEN` | *(auto-generated)* | Shared auth token between dragonpbx and the UI |
| `DRACHTIO_SECRET` | `cymru` | Drachtio admin secret |
| `CONFIG_URL` | - | URL for the config API (set to the UI service) |
| `CONFIG_TTL` | `60` | Config cache TTL in seconds |
| `DATA_SOURCE` | `api` | Config data source type |
| `SESSION_SECRET` | `changeme` | Sails session secret for the UI |
| `LOGLEVEL` | `info` | Log level for dragonpbx |

## Shared Volumes

The `media` and `media-cache` volumes are shared between dragonpbx and rtpengine so that audio files (announcements, prompts, etc.) are accessible to both the application and the media proxy.

| Volume | Mount Point | Purpose |
|--------|-------------|---------|
| `media` | `/app/media` | Static audio files |
| `media-cache` | `/tmp` | Downloaded/cached media |
| `ui-data` | `/app/.tmp` | UI database (sails-disk) |

## Files

- `Dockerfile` - Builds dragonpbx from GitHub
- `Dockerfile.ui` - Builds dragonpbx-ui from GitHub
- `docker-compose.yml` - Service definitions
- `sails-production.js` - Sails config overrides for the UI (sockets, tokens)
- `start.sh` - Startup script with auto token generation
